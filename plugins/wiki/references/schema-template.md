# Schema Template (CLAUDE.md)

This is the default CLAUDE.md generated by `/wiki:init`. It serves as Layer 3 of the Karpathy LLM Wiki pattern — the schema that tells the LLM how the wiki is structured.

---

```markdown
# Wiki Schema

This is an LLM-maintained knowledge base following [Karpathy's LLM Wiki pattern](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f).

## Three-Layer Architecture

1. **raw/** — Immutable source documents. Never modify files in this directory.
2. **wiki/** — LLM-compiled pages. You maintain these pages.
3. **This file (CLAUDE.md)** — Schema and conventions. Update only when conventions change.

## Directory Structure

```
raw/
  articles/          — Web articles, blog posts
  papers/            — Research papers, reports
  assets/            — Images, diagrams, data files
wiki/
  index.md           — Content catalog (always keep updated)
  log.md             — Append-only operations changelog
  entities/          — People, organizations, products
  concepts/          — Ideas, frameworks, patterns, technologies
  sources/           — Summaries of ingested raw documents
  synthesis/         — Cross-cutting analysis and comparisons
```

## Page Conventions

### Frontmatter (required on every page)

```yaml
---
type: entity | concept | source | synthesis
created: YYYY-MM-DD
updated: YYYY-MM-DD
sources: ["[[sources/source-name]]"]
tags:
  - wiki/<type>
  - <domain-tag>
---
```

### Wikilinks

- Use `[[page-name]]` for all cross-references
- Use `[[page-name|display text]]` when the page name is not human-readable
- Every page must have at least one outbound link
- A page without inbound links is an orphan — fix by linking from related pages

### Naming

- File names: kebab-case (e.g., `andrej-karpathy.md`, `transformer-architecture.md`)
- One concept per page — if a page covers multiple distinct ideas, split it
- Prefer specific names over generic ones (`attention-mechanism.md` not `attention.md`)

## Page Type Guidelines

### Entity (`entities/`)
- Facts, roles, affiliations, timeline
- Link to: concepts they're associated with, sources that mention them
- Update when new information arrives

### Concept (`concepts/`)
- Definition, explanation, examples
- Link to: related concepts, entities who work on it, sources that discuss it
- Include practical implications, not just definitions

### Source (`sources/`)
- Summary of the raw document (3-5 key takeaways)
- Metadata: author, date, URL (if applicable)
- Link to: entities mentioned, concepts discussed
- Include notable quotes

### Synthesis (`synthesis/`)
- Cross-cutting analysis that connects multiple sources/concepts
- Comparisons, trend analysis, evolved understanding
- Link to: all sources, entities, and concepts referenced
- These are the most valuable pages — they represent accumulated insight

## Operations

### Ingest
When adding a source:
1. Save raw document to `raw/` (appropriate subdirectory)
2. Create/update `wiki/sources/<source-name>.md`
3. Create/update entity pages for people/orgs mentioned
4. Create/update concept pages for key ideas
5. Update `wiki/index.md` — add entries for new pages
6. Append to `wiki/log.md`: `## [YYYY-MM-DD] ingest | <Source Title>`
7. Check for synthesis opportunities — does this connect to existing knowledge?

### Query
When answering questions:
1. Search `wiki/index.md` for relevant pages
2. Read relevant pages, following wikilinks
3. Synthesize answer with `[[citations]]`
4. If the answer reveals novel connections, create a synthesis page

### Lint
Periodic health checks:
1. Find orphan pages (no inbound links)
2. Find contradictions between pages
3. Find important concepts without dedicated pages
4. Find missing cross-references
5. Check for stale information (sources updated but wiki not)
6. Report findings and suggest fixes

## Index Format

`wiki/index.md` is a content catalog. Each page listed with:
- Wikilink
- One-line summary
- Organized by category (entities, concepts, sources, synthesis)
- Dataview queries for dynamic views

## Log Format

`wiki/log.md` is append-only. Each entry:

```markdown
## [YYYY-MM-DD] <operation> | <Title>

<Brief description of what was done. Pages created/updated.>
```

Operations: `ingest`, `query`, `lint`, `update`, `synthesis`
```
